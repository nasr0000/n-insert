Привет! Нам нужно разработать Chrome Extension (расширение для браузера), которое будет работать в мобильном браузере (Kiwi/Yandex) поверх CRM-системы AinurPOS (URL: web.ainur.app).

### Суть задачи:
В интерфейсе создания закупки в AinurPOS есть зона «Выберите товары или нажмите ctrl+v для вставки из Excel» (это интерактивный div, перехватывающий событие 'paste'). На смартфонах вызвать Ctrl+V штатными средствами невозможно. Расширение должно решить эту проблему, добавив плавающую UI-кнопку "Вставить товары", которая программно имитирует событие вставки данных.

### Стек технологий:
- Frontend расширения: Manifest V3 (JavaScript, HTML, CSS)
- Backend & База данных: Google Firebase (Firebase Auth, Firestore, Firebase Hosting)

### Архитектура и функционал:

1. **Интеграция Firebase:**
   - Подключи Firebase SDK в context расширения (через background script / service worker или напрямую в content script, если позволяет политика безопасности контента).
   - **Firestore:** Используется для хранения истории вставок, логов ошибок и, при необходимости, кастомных шаблонов сопоставления колонок (артикул, количество, цена).
   - **Firebase Auth:** Анонимная авторизация (Anonymous Auth) или авторизация через Google, чтобы привязать историю вставок к конкретному пользователю/устройству.
   - **Firebase Hosting:** Если расширению понадобится полноценная веб-панель управления или аналитики, предусмотри конфигурацию для деплоя статики.

2. **UI Расширения (Content Script):**
   - Рендеринг плавающей (fixed) кнопки "📋 Вставить товары" в правом нижнем углу страницы `web.ainur.app/card/docs/purchase`.
   - Кнопка должна быть адаптирована под тач-интерфейс смартфона (крупная, с хорошими отступами, z-index: 99999).
   - При клике на кнопку запрашивается доступ к системному буферу обмена (`navigator.clipboard.readText()`).

3. **Логика эмуляции Ctrl+V:**
   - Скрипт должен создавать валидный `ClipboardEvent('paste')` с заполнением `clipboardData` текстовыми данными из буфера.
   - Событие должно отправляться на `document` и на текущий активный элемент (`document.activeElement`), чтобы AinurPOS корректно перехватил строку данных (обычно это TSV/CSV формат из Excel).

4. **Логирование:**
   - После успешной вставки отправляй в Firestore документ: `{ userId, timestamp, textLength, status: "success" }`.

### Что нужно сделать прямо сейчас:
1. Напиши структуру файлов проекта: `manifest.json`, `content.js`, `background.js`, `firebase-config.js`.
2. Напиши код `manifest.json` с разрешениями `"permissions": ["clipboardRead", "activeTab", "storage"]` и настроенными правилами для работы с Firebase CDN или локальными файлами SDK.
3. Реализуй отказоустойчивый скрипт эмуляции события Paste, который гарантированно триггерит обработчик на веб-странице.
